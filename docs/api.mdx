## Core Types

### DB

**Source**: `/sqlx.go` (line ~243)

The DB type wraps `sql.DB` and provides enhanced functionality.

```go
type DB struct {
    *sql.DB
    // contains filtered or unexported fields
}
```

#### Connection Functions

<ResponseField name="Connect" type="func(driverName, dataSourceName string) (*DB, error)">
  Opens a database connection and pings it to verify connectivity.
  
  ```go
  db, err := sqlx.Connect("postgres", "user=foo dbname=bar")
  ```
</ResponseField>

<ResponseField name="Open" type="func(driverName, dataSourceName string) (*DB, error)">
  Opens a database handle without verifying connection (lazy connection).
  
  ```go
  db, err := sqlx.Open("postgres", "user=foo dbname=bar")
  ```
</ResponseField>

<ResponseField name="MustConnect" type="func(driverName, dataSourceName string) *DB">
  Like Connect but panics on error.
  
  ```go
  db := sqlx.MustConnect("postgres", "user=foo dbname=bar")
  ```
</ResponseField>

<ResponseField name="MustOpen" type="func(driverName, dataSourceName string) *DB">
  Like Open but panics on error.
  
  ```go
  db := sqlx.MustOpen("postgres", "user=foo dbname=bar")
  ```
</ResponseField>

#### Query Methods

<ResponseField name="Get" type="func(dest interface{}, query string, args ...interface{}) error">
  Executes a query and scans a single row into dest.
  
  ```go
  var user User
  err := db.Get(&user, "SELECT * FROM users WHERE id=$1", 1)
  ```
</ResponseField>

<ResponseField name="GetContext" type="func(ctx context.Context, dest interface{}, query string, args ...interface{}) error">
  Context-aware version of Get.
  
  ```go
  ctx := context.Background()
  var user User
  err := db.GetContext(ctx, &user, "SELECT * FROM users WHERE id=$1", 1)
  ```
</ResponseField>

<ResponseField name="Select" type="func(dest interface{}, query string, args ...interface{}) error">
  Executes a query and scans multiple rows into dest slice.
  
  ```go
  var users []User
  err := db.Select(&users, "SELECT * FROM users")
  ```
</ResponseField>

<ResponseField name="SelectContext" type="func(ctx context.Context, dest interface{}, query string, args ...interface{}) error">
  Context-aware version of Select.
  
  ```go
  ctx := context.Background()
  var users []User
  err := db.SelectContext(ctx, &users, "SELECT * FROM users")
  ```
</ResponseField>

<ResponseField name="Queryx" type="func(query string, args ...interface{}) (*Rows, error)">
  Executes a query and returns sqlx.Rows for manual iteration.
  
  ```go
  rows, err := db.Queryx("SELECT * FROM users")
  defer rows.Close()
  ```
</ResponseField>

<ResponseField name="QueryxContext" type="func(ctx context.Context, query string, args ...interface{}) (*Rows, error)">
  Context-aware version of Queryx.
</ResponseField>

<ResponseField name="QueryRowx" type="func(query string, args ...interface{}) *Row">
  Executes a query expected to return at most one row.
  
  ```go
  row := db.QueryRowx("SELECT * FROM users WHERE id=$1", 1)
  ```
</ResponseField>

<ResponseField name="QueryRowxContext" type="func(ctx context.Context, query string, args ...interface{}) *Row">
  Context-aware version of QueryRowx.
</ResponseField>

#### Transaction Methods

<ResponseField name="Beginx" type="func() (*Tx, error)">
  Begins a transaction with sqlx extensions.
  
  ```go
  tx, err := db.Beginx()
  if err != nil {
      log.Fatal(err)
  }
  defer tx.Rollback()
  ```
</ResponseField>

<ResponseField name="BeginTxx" type="func(ctx context.Context, opts *sql.TxOptions) (*Tx, error)">
  Begins a transaction with context and options.
  
  ```go
  ctx := context.Background()
  tx, err := db.BeginTxx(ctx, &sql.TxOptions{Isolation: sql.LevelSerializable})
  ```
</ResponseField>

<ResponseField name="MustBegin" type="func() *Tx">
  Like Beginx but panics on error.
  
  ```go
  tx := db.MustBegin()
  ```
</ResponseField>

#### Connection Methods

<ResponseField name="Connx" type="func(ctx context.Context) (*Conn, error)">
  Returns a single connection from the pool (added in v1.3.0).
  
  ```go
  conn, err := db.Connx(context.Background())
  defer conn.Close()
  ```
</ResponseField>

#### Utility Methods

<ResponseField name="DriverName" type="func() string">
  Returns the driver name used by the database.
  
  ```go
  driver := db.DriverName() // e.g., "postgres"
  ```
</ResponseField>

<ResponseField name="Rebind" type="func(query string) string">
  Converts query to use the driver's bindvar type.
  
  ```go
  query := db.Rebind("SELECT * FROM users WHERE id=?")
  // PostgreSQL: "SELECT * FROM users WHERE id=$1"
  ```
</ResponseField>

<ResponseField name="BindNamed" type="func(query string, arg interface{}) (string, []interface{}, error)">
  Binds a named query to the driver's bindvar type and extracts arguments.
  
  ```go
  query, args, err := db.BindNamed("SELECT * FROM users WHERE name=:name", user)
  ```
</ResponseField>

<ResponseField name="Unsafe" type="func() *DB">
  Returns a version of DB with unsafe mode enabled (disables column count matching).
  
  ```go
  unsafeDB := db.Unsafe()
  ```
</ResponseField>

<ResponseField name="MapperFunc" type="func(mf func(string) string) *DB">
  Sets a custom name mapping function for this DB instance.
  
  ```go
  db = db.MapperFunc(strings.ToUpper)
  ```
</ResponseField>

### Rows

**Source**: `/sqlx.go` (line ~577)

Enhanced result set with struct scanning capabilities.

```go
type Rows struct {
    *sql.Rows
    // contains filtered or unexported fields
}
```

#### Methods

<ResponseField name="StructScan" type="func(dest interface{}) error">
  Scans the current row into a struct.
  
  ```go
  var user User
  for rows.Next() {
      err := rows.StructScan(&user)
      if err != nil {
          log.Fatal(err)
      }
      fmt.Println(user)
  }
  ```
</ResponseField>

<ResponseField name="MapScan" type="func(dest map[string]interface{}) error">
  Scans the current row into a map.
  
  ```go
  results := make(map[string]interface{})
  for rows.Next() {
      err := rows.MapScan(results)
      if err != nil {
          log.Fatal(err)
      }
      fmt.Println(results)
  }
  ```
</ResponseField>

<ResponseField name="SliceScan" type="func() ([]interface{}, error)">
  Scans the current row into a slice.
  
  ```go
  for rows.Next() {
      values, err := rows.SliceScan()
      if err != nil {
          log.Fatal(err)
      }
      fmt.Println(values)
  }
  ```
</ResponseField>

<ResponseField name="Columns" type="func() ([]string, error)">
  Returns column names (inherited from sql.Rows).
</ResponseField>

<ResponseField name="ColumnTypes" type="func() ([]*sql.ColumnType, error)">
  Returns column types (inherited from sql.Rows).
</ResponseField>

<ResponseField name="Err" type="func() error">
  Returns error from iteration (inherited from sql.Rows).
</ResponseField>

<ResponseField name="Next" type="func() bool">
  Advances to next row (inherited from sql.Rows).
</ResponseField>

<ResponseField name="Close" type="func() error">
  Closes the rows (inherited from sql.Rows).
</ResponseField>

## Common Error Handling

All sqlx functions return standard Go errors. Common errors:

- `sql.ErrNoRows` - Query returned no rows (from Get or QueryRow)
- `sql.ErrTxDone` - Transaction has already been committed or rolled back
- `sql.ErrConnDone` - Connection has already been closed

Example error handling:

```go
var user User
err := db.Get(&user, "SELECT * FROM users WHERE id=$1", id)
if err != nil {
    if err == sql.ErrNoRows {
        return nil, errors.New("user not found")
    }
    return nil, fmt.Errorf("database error: %w", err)
}
```
