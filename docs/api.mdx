## Overview

This page provides comprehensive API documentation for the @zxing/library. The library exports are organized into several categories:

- **Browser APIs** - Camera and video scanning
- **Core Readers** - Barcode decoding
- **Core Writers** - Barcode generation
- **Data Structures** - Image processing classes
- **Configuration** - Enums and hint types
- **Exceptions** - Error types

## Browser APIs

### BrowserCodeReader

Abstract base class for all browser-based barcode readers. Provides camera access and video decoding capabilities.

**Import:**
```typescript
import { BrowserCodeReader } from '@zxing/library';
```

**Location:** `/src/browser/BrowserCodeReader.ts`

#### Methods

##### listVideoInputDevices()

Lists all available video input devices (cameras).

```typescript
async listVideoInputDevices(): Promise<MediaDeviceInfo[]>
```

**Returns:** Array of `MediaDeviceInfo` objects describing available cameras.

**Example:**
```typescript
const devices = await codeReader.listVideoInputDevices();
devices.forEach(device => {
  console.log(device.label, device.deviceId);
});
```

##### decodeFromVideoDevice()

Continuously decode from a video device (camera).

```typescript
async decodeFromVideoDevice(
  deviceId: string | undefined,
  videoElement: string | HTMLVideoElement,
  callbackFn: DecodeContinuouslyCallback
): Promise<void>
```

**Parameters:**
- `deviceId` - Camera device ID (undefined for default camera)
- `videoElement` - Video element ID or HTMLVideoElement
- `callbackFn` - Callback invoked for each decode attempt

**Callback Signature:**
```typescript
(result: Result | undefined, error: Exception | undefined) => void
```

##### decodeFromInputVideoDevice()

Decode once from a video device.

```typescript
async decodeFromInputVideoDevice(
  deviceId: string | undefined,
  videoElementId: string
): Promise<Result>
```

**Returns:** Promise resolving to decoded `Result`.

##### decodeFromImageElement()

Decode from an HTML image element.

```typescript
async decodeFromImageElement(
  imageElement: string | HTMLImageElement
): Promise<Result>
```

##### decodeFromImageUrl()

Decode from an image URL.

```typescript
async decodeFromImageUrl(imageUrl: string): Promise<Result>
```

##### decodeFromVideoElement()

Decode once from a video element.

```typescript
async decodeFromVideoElement(
  videoElement: string | HTMLVideoElement
): Promise<Result>
```

##### reset()

Stop scanning and release camera resources.

```typescript
reset(): void
```

### BrowserMultiFormatReader

Reads multiple barcode formats from browser sources.

**Import:**
```typescript
import { BrowserMultiFormatReader } from '@zxing/library';
```

**Location:** `/src/browser/BrowserMultiFormatReader.ts`

**Extends:** `BrowserCodeReader`

#### Constructor

```typescript
constructor(
  hints?: Map<DecodeHintType, any>,
  timeBetweenScansMillis?: number
)
```

**Parameters:**
- `hints` - Decode hints to configure scanning behavior
- `timeBetweenScansMillis` - Time between decode attempts (default: 500ms)

**Example:**
```typescript
const hints = new Map();
hints.set(DecodeHintType.POSSIBLE_FORMATS, [
  BarcodeFormat.QR_CODE,
  BarcodeFormat.EAN_13
]);

const reader = new BrowserMultiFormatReader(hints, 300);
```

Main reader supporting multiple barcode formats.

**Import:**
```typescript

```

**Location:** `/src/core/MultiFormatReader.ts`

**Implements:** `Reader`

#### Methods

##### decode()

Decode a barcode from a binary bitmap.

```typescript
decode(
  image: BinaryBitmap,
  hints?: Map<DecodeHintType, any>
): Result
```

##### decodeWithState()

Decode using cached hints (faster for continuous scanning).

```typescript
decodeWithState(image: BinaryBitmap): Result
```

##### setHints()

Configure reader behavior.

```typescript
setHints(hints: Map<DecodeHintType, any>): void
```

##### reset()

Reset internal state.

```typescript
reset(): void
```

**Example:**
```typescript
const reader = new MultiFormatReader();
const hints = new Map();
hints.set(DecodeHintType.TRY_HARDER, true);
reader.setHints(hints);

const result = reader.decodeWithState(binaryBitmap);
console.log(result.getText());
```

### DecodeHintType

Configuration hints for decoders.

**Import:**
```typescript
import { DecodeHintType } from '@zxing/library';
```

**Location:** `/src/core/DecodeHintType.ts`

#### Values

```typescript
enum DecodeHintType {
  OTHER,                           // Application-specific hint
  PURE_BARCODE,                    // Clean barcode image
  POSSIBLE_FORMATS,                // Restrict to specific formats
  TRY_HARDER,                      // More thorough scanning
  CHARACTER_SET,                   // Character encoding
  ALLOWED_LENGTHS,                 // Valid barcode lengths
  ASSUME_CODE_39_CHECK_DIGIT,      // Code 39 check digit
  ENABLE_CODE_39_EXTENDED_MODE,    // Code 39 extended mode
  ASSUME_GS1,                      // GS1 formatting
  RETURN_CODABAR_START_END,        // Include Codabar delimiters
  NEED_RESULT_POINT_CALLBACK,      // ResultPoint callback
  ALLOWED_EAN_EXTENSIONS           // EAN extension lengths
}
```

**Example:**
```typescript
const hints = new Map();
hints.set(DecodeHintType.TRY_HARDER, true);
hints.set(DecodeHintType.POSSIBLE_FORMATS, [BarcodeFormat.QR_CODE]);
hints.set(DecodeHintType.CHARACTER_SET, 'UTF-8');
```

## Exception Types

All exceptions located at `/src/core/*Exception.ts`.

### Exception

Base exception class.

**Import:** `import { Exception } from '@zxing/library';`

### ReaderException

Base for all reading errors.

**Import:** `import { ReaderException } from '@zxing/library';`

### NotFoundException

Barcode not found in image.

**Import:** `import { NotFoundException } from '@zxing/library';`

### FormatException

Invalid barcode format.

**Import:** `import { FormatException } from '@zxing/library';`

### ChecksumException

Checksum validation failed.

**Import:** `import { ChecksumException } from '@zxing/library';`

### WriterException

Error during encoding.

**Import:** `import { WriterException } from '@zxing/library';`

### Other Exceptions

- `ArgumentException` - Invalid arguments
- `IllegalArgumentException` - Illegal argument value
- `IllegalStateException` - Invalid state
- `UnsupportedOperationException` - Unsupported operation
- `ArithmeticException` - Math errors
- `ReedSolomonException` - Error correction failed
